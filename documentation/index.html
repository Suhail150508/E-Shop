Single Vendor eCommerce Documentation

Last Updated: February 2026

Welcome

Answer:

Thank you for choosing Single Vendor eCommerce with Virtual Try-On. This documentation helps you install, configure, and use the system. It also explains how customers can place orders, use the wallet, and request refunds—so you can share the relevant parts with your buyers.

Note: If you find any issue not covered here, please contact our support team.

How it Works

Answer:

The system is a single-vendor online store: one admin runs the entire shop. Customers browse products on the frontend, add items to the cart, and complete checkout (shipping address → payment → confirmation). You manage everything from the Admin Panel: products, orders, refunds, customers, wallet, support tickets, live chat, and settings. Optional features include Virtual Try-On (e.g. sunglasses), Wallet (balance, pay at checkout), Multi-Currency, and Multi-Language with RTL support.

Screenshot: Home page with hero, categories, and product sections

Key Features

Answer:
  • Image-based Virtual Try-On (no AI required) for suitable products (e.g. glasses).
  • Wallet – customers can hold balance and pay at checkout; refunds can be credited to wallet.
  • Multiple payment gateways – Stripe, PayPal, Razorpay, Paystack, Bank Transfer, Cash on Delivery.
  • Multi-Currency & Multi-Language with RTL support (e.g. Arabic).
  • Support Tickets and Live Chat for customer support.
  • Refund system – customers request refund with a reason; admin approves or rejects.
  • Manage Pages – About, Contact, Terms, Privacy, Shipping (editable from admin).
  • Website Setup – home page content, auth page titles/images, shop labels from one place.
  • Interactive Map – Integrated OpenStreetMap (Leaflet) for address selection. Free to use & No API Key required.

Server Requirements

Answer:

To run the system properly, ensure your server meets:

  • PHP >= 8.2
  • Database – MySQL 5.7+ / MariaDB 10.3+ or PostgreSQL or SQLite
  • PHP extensions: BCMath, Ctype, Fileinfo, JSON, Mbstring, OpenSSL, PDO, Tokenizer, XML
  • GD or Imagick (for images and Virtual Try-On)
  • Composer 2.x, Node.js & NPM (for frontend assets)

How to Complete an Order?

Answer:
  1. Browse – Open the store, go to Shop or a Category, and click a product to view details.
  2. Add to Cart – Choose quantity (and options like color/size if available), then click Add to Cart.
  3. Cart – Click the cart icon or go to Cart. Review items, change quantity if needed, then click Checkout (login/register if required).
  4. Shipping – Select or add a delivery address. Choose Home Delivery or Pick Up. Add an order note if you want. Click Proceed to Payment.
  5. Payment – Select a payment method (Card, PayPal, COD, Wallet, etc.). Optionally apply a Coupon. Click Pay Now or Place Order.
  6. Confirmation – You will see an order success page with order number. You can go to Account → My Orders to view the order and download the invoice.

Using the Interactive Map

Answer:

The system features a powerful Interactive Map to simplify address management for customers.

  • Pin Exact Location: Customers can drag and drop a pin on the map to set their precise delivery location.
  • Auto-Address Fill: Clicking on the map automatically fills in address fields (Street, City, Zip Code) using reverse geocoding.
  • Zero Configuration: Uses OpenStreetMap (Leaflet), so it is 100% Free and requires No API Keys (unlike Google Maps).
  • Visual Confirmation: Ensures the delivery address is accurate, reducing failed deliveries.

How to Use Wallet?

Answer:

The Wallet holds your store balance. You can use it to pay for orders at checkout when the balance is sufficient.

  • Log in and go to Account (or the Wallet/Balance link if available in the menu).
  • View your current balance and recent transactions.
  • At Checkout, if Wallet is enabled, you will see Pay with Wallet as an option. Select it to pay using your balance (full or partial, depending on store settings).
Screenshot: Customer account – Wallet balance and transactions

How to Deposit to Wallet?

Answer:

Whether customers can deposit (add money) to the wallet depends on your store setup. If the admin has enabled a “Top Up” or “Add Funds” feature:

  • Go to Account → Wallet (or similar).
  • Click Add Funds or Deposit.
  • Enter the amount and complete payment via the available gateway (e.g. Stripe, PayPal). The amount will be credited to your wallet after successful payment.

If you do not see a deposit option, your store may only use the wallet for refunds (admin credits refunds to the wallet). Check with the store owner.

How to Request a Refund?

Answer:
  1. Log in and go to Account → My Orders.
  2. Open the order you want to refund.
  3. Click Request Refund (or similar).
  4. Select a reason from the list (e.g. Damaged product, Wrong item) and add any message or proof if asked.
  5. Submit the request. The admin will review and either Approve or Reject. You may receive the refund to your Wallet or via the original payment method, depending on store policy.
Screenshot: Order detail page with Request Refund button

When Can You Request a Refund?

Answer:

Typically you can request a refund from the order detail page in your account after the order is placed. The exact rules (e.g. within how many days, for which order statuses) are set by the store. Common cases:

  • Order is Pending or Processing – you may request cancellation/refund.
  • Order is Shipped or Delivered – you may still request refund (e.g. damaged item, wrong product) and the admin will decide.

If you do not see Request Refund, the store may have disabled it for that order or your account. Contact support for clarification.

Refund Policy

Answer:

Refund policy is defined by the store owner. In this system:

  • Customers submit a refund request with a reason (from the list configured by admin).
  • The admin reviews each request and can Approve or Reject it.
  • If approved, the refund may be credited to the customer’s Wallet or processed via the original payment method (e.g. Stripe, PayPal) – this depends on how the admin handles it.
  • Store-specific rules (time limits, eligible products, restocking fees) should be published on the store’s Terms or Refund Policy page (editable via Admin → Manage Pages).

How to install on Local Server?

Answer:

Follow these steps to install the project on your local machine (XAMPP/WAMP):

  1. Download the codecanyon-item.zip file and extract it.
  2. Copy the project folder to your htdocs (XAMPP) or www (WAMP) directory.
  3. Open phpMyAdmin and create a new database (e.g., ecommerce_db).
  4. Rename the .env.example file to .env.
  5. Open the terminal in the project directory and run:
composer install
php artisan key:generate
php artisan migrate --seed
php artisan storage:link
php artisan serve

How to install on Live Server (cPanel)?

Answer:

To install on a live server, follow these instructions:

  1. Log in to your cPanel.
  2. Go to File Manager and upload the project ZIP file to public_html or your subdomain folder.
  3. Extract the ZIP file.
  4. Create a new Database and User from MySQL Database Wizard.
  5. Import the database.sql file (provided in the root folder) into your database via phpMyAdmin.
  6. Edit the .env file and update the database credentials.

How to upgrade the system?

Answer:

To upgrade to a newer version of the software, please follow these steps carefully:

  1. Backup: Always backup your files and database before upgrading.
  2. Upload: Download the latest version from CodeCanyon and replace the existing files on your server (excluding the .env file and public/storage folder).
  3. Update Dependencies: If there are changes in dependencies, run composer install.
  4. Migrate: Run php artisan migrate to update the database structure.
  5. Clear Cache: Run php artisan optimize:clear to clear all caches.

How to manage Menus (Header & Footer)?

Answer:

The Menu Builder allows you to fully customize the navigation links on your website's header and footer without touching any code.

1. Available Menu Locations

  • Header: The main navigation menu at the top of the site (supports dropdowns).
  • Footer 1, 2, 3, 4: Four distinct columns in the footer area for links like "Quick Links", "Company", etc.

2. Creating/Editing a Menu

  1. Go to Content & Settings > Menus.
  2. To create a new menu, click Create Menu, enter a name (e.g., "Main Header"), and select its Position (e.g., "Header").
  3. To edit an existing menu, click the Builder button next to the menu name.

3. Using the Menu Builder

In the Builder interface, you can add different types of items:

  • Pages: Select from your existing dynamic pages (e.g., About Us, Contact).
  • Custom Links: Enter any URL and Link Text (useful for external links).
  • Categories: (If available) Link directly to product categories.

Reordering & Nesting:

  • Drag and Drop items to reorder them.
  • Create Dropdowns: Drag an item slightly to the right under another item to make it a "child" (sub-menu) of that item. This creates a dropdown menu on the frontend.
Tip: Don't forget to click the "Save Menu" button after making changes to the structure!

How to manage General Settings & Delivery Charges?

Answer:

This section controls core site configurations including branding and shipping logic.

  1. Go to System Settings > General Settings.
  2. App Settings: Change App Name, Currency Symbol, and Timezone.
  3. Logo & Favicon: Upload your brand's Header/Footer Logo and Favicon.
  4. Shipping Configuration (Delivery Charges):
    • Inside City Name: Define the main city (e.g., "Dhaka"). If a customer's address matches this city, the "Inside City" rate applies.
    • Inside City Cost: Set the delivery charge for the main city.
    • Outside City Cost: Set the delivery charge for all other locations (any city not matching the "Inside City Name").
    • Free Shipping Minimum Amount: Set a minimum order subtotal (e.g., 5000) to automatically offer Free Shipping. Enter 0 to disable this feature.
  5. Footer Information: Update contact details (Phone, Email, Address), copyright text, and payment method image.
  6. Social Media Links: Configure links for Facebook, Twitter, Instagram, etc.
